使用 git-flow 進行高效率的分支模式。 by Vincent Driessen

關於

git-flow 是一個 git 的延伸工具，按 Vincent Driessen 的分支模型提供高層次的儲存庫操作。 查看詳情

★ ★ ★

這個速記表展示了 git-flow 的基本操作和效果。

★ ★ ★

基礎建議

  • Git flow 提供了極出色的指令使用說明以及輸出提示。請仔細閱讀並觀察發生了什麼事情...
  • OSX 程序 Sourcetree 是一個極出色的 git 圖形用戶界面，已經提供了 git-flow 的支持。
  • - Git-flow 是一個基於歸併的解決方案，它並沒有提供重置(rebase)分支的能力。

★ ★ ★

安裝

  • 你需要有一個可以正常運作的 git 作為前提。
  • Git flow 可以在 OSX, Linux 和 Windows之下使用

★ ★ ★

OSX

$ brew install git-flow

Linux

$ apt-get install git-flow

Windows (Cygwin)

$ wget -q -O - --no-check-certificate https://github.com/nvie/gitflow/raw/develop/contrib/gitflow-installer.sh | bash

安裝 git-flow, 你需要 wget 和 util-linux。

更多的 git flow 安裝指引，請閱讀 git flow wiki.

開始

為了自定義你的項目，Git flow 需要初始化過程。

★ ★ ★

初始化

使用 git-flow，從初始化一個現有的 git 庫內開始:

git flow init

你必須回答幾個關於分支的命名約定的問題。
建議使用默認值。

特性

  • 為即將發佈的版本開發新功能特性。
  • 這通常只存在開發者的庫中。

★ ★ ★

增加新特性

新特性的開發是基於 'develop' 分支的。

通過下面的指令開始開發新特性：

git flow feature start MYFEATURE

這個操作創建了一個基於'develop'的特性分支，並切換到這個分支之下。

完成新特性

完成開發新特性。這個動作執行下面的操作.

  • 合併 MYFEATURE 分支到 'develop'
  • 刪除這個新特性分支
  • 切換回 'develop' 分支
git flow feature finish MYFEATURE

發佈新特性

你是否合作開發一項新特性？
發佈新特性分支到遠程服務器，所以，其它用戶也可以使用這分支。

git flow feature publish MYFEATURE

取得一個發佈的新特性分支

取得其它用戶發佈的新特性分支，並簽出遠程的變更。

git flow feature pull MYFEATURE

作一個release版本

  • 支持一個新的用於生產環境的發佈版本。
  • 允許修正小問題，並為發佈版本準備元數據。

★ ★ ★

開始準備release版本

開始準備release版本，使用 git flow release 指令.

它從 'develop' 分支開始創建一個 release 分支。

git flow release start RELEASE [BASE]

你可以選擇提供一個 [BASE]參數，即提交記錄的 sha-1 hash 值，來開啟動 release 分支. 這個提交記錄的 sha-1 hash 值必須是'develop' 分支下的。

★ ★ ★

創建 release 分支之後立即發佈允許其它用戶向這個 release 分支提交見容是個明智的做法。指令十分類似發佈新特性：

git flow release publish RELEASE

(你可以通過
git flow release track RELEASE 指令簽出 release 版本的遠程變更)

完成 release 版本

完成 release 版本是一個大 git 分支操作。它執行下面幾個動作：

  • 歸併 release 分支到 'master' 分支
  • 用 release 分支名打 Tag
  • 歸併 release 分支到 'develop'
  • 移除 release 分支
git flow release finish RELEASE

熱點修復

  • 熱點修復來自這樣的需求：生產環境的版本處於一個不預期狀態，需要立即修正。
  • 有可能是需要修正 master 分支上某個 TAG 標記的生產版本。

★ ★ ★

開始 git flow 熱點修復

像其它 git flow 指令一樣, 熱點修復分支開始自：

git flow hotfix start VERSION [BASENAME]

VERSION 參數標記著修正版本。你可以從 [BASENAME]開始

完成熱點修復

當完成熱點分支，代碼歸併回 develop 和 master 分支。相應地，master 分支打上修正版本的 TAG。

git flow hotfix finish VERSION
